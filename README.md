# play-mjm2026cup-guide

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常与特定主题或活动相关，以静态网页形式呈现，便于直接浏览或部署到各类托管服务。

## 目录说明

```
├── index.html          # 入口页面（如存在）
├── pages/              # 存放各个独立 HTML 页面的文件夹
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：收录所有独立的 HTML 页面，每个页面可独立运行或访问。
- `assets/`：存放页面可能用到的公共资源，如样式表、脚本和图片。
- `index.html`：可选入口文件，用于展示页面列表或导航。

## 页面归档说明

本仓库中的 HTML 页面均为静态文件，无需后端支持。你可以：

- 直接通过 GitHub Pages 或其他静态托管服务部署整个仓库。
- 将单个 HTML 文件下载到本地，用浏览器打开即可使用。
- 按需引用 `assets/` 目录下的资源文件。

页面内容可能包含演示、说明或工具界面，具体用途请参考各页面内的注释或标题。

## 维护说明

- 添加新页面：将 HTML 文件放入 `pages/`，如有需要可同步更新 `assets/` 中的资源。
- 更新页面：直接修改对应文件，保持文件结构清晰。
- 删除页面：移除对应文件，并检查是否有其他页面引用其资源。
- 建议为每个页面在文件头部添加简明注释，说明其用途和依赖。

## 其他

本仓库仅作为 HTML 页面的存档与分发用途。如有问题或建议，欢迎提交 Issue 或 Pull Request。
